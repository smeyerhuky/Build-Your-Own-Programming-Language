# x86-64 (subset used by Ch 13)

The Ch 13 back-end emits **AT&T-syntax** x86-64 assembly. Every
instruction is one `x64` record:

```java
class x64 {
  String op;
  x64loc opnd1, opnd2;
  // op opnd1, opnd2     ── two-operand form
  // op opnd1            ── one-operand form
  // op                  ── zero-operand form
  // op == "lab" prints "<label>:"
}
```

## Operand modes (`x64loc.mode`, from `/ch13/x64loc.java`)

| Mode | Built by | Printed as | Example |
|---|---|---|---|
| 1 | `x64loc(String r)` | `r` | `%rax` |
| 2 | `x64loc(int i)` | `i` | `8` |
| 3 | `x64loc("reg", off)` | `off(reg)` | `-8(%rbp)` |
| 4 | `x64loc("reg", "label")` | `label(reg)` | `.L1(%rip)` |
| 5 | `x64loc("imm", n)` | `$n` | `$42` |
| 6 | `x64loc("lab", n)` | `.Ln` or `label` | `.L7` |

## Calling convention

System V AMD64 (Linux/macOS):

- Args: `%rdi, %rsi, %rdx, %rcx, %r8, %r9`.
- Return: `%rax`.
- Callee-save: `%rbx, %rbp, %r12, %r13, %r14, %r15`.
- Caller-save: everything else.

## Standard prologue/epilogue

```
foo:
    pushq  %rbp
    movq   %rsp, %rbp
    subq   $N, %rsp     # N bytes of locals
    ...
    movq   %rbp, %rsp
    popq   %rbp
    ret
```

## Labels

`.L<n>` for compiler-generated labels (jumps, string-table entries).
The label-emission path is `x64("lab", x64loc)` which prints
`label:` with no leading tab.
