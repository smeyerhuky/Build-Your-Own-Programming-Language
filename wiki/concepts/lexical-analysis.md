# Lexical analysis

A **lexer** (a.k.a. scanner) reads source characters and emits **tokens**
— `(category, lexeme, line, column)` tuples. Regular expressions
describe each token category; the longest match wins, with the rule
declared earliest winning a tie (the *longest-match*, then
*priority* rule). Whitespace and comments are recognized but discarded.

In this course the lexer is generated by **Flex** (Java target,
*JFlex* under the hood) or **Uflex** (Unicon target) from a `.l` file —
see [`/wiki/tools/flex-uflex.md`](/wiki/tools/flex-uflex.md).

## Where it appears in the book

- First introduced in [Ch 3 — Scanning Source Code](/wiki/chapters/ch03-lexical-analysis/index.md).
- Re-used and extended in every later chapter via `javalex.l`.

## Anchor file

`/ch3/javalex.l` is the canonical J0 lexer specification. Its three
sections (definitions, rules, code) are the smallest end-to-end Flex
file in the repo.

## Key invariants

- Reserved words must be listed **before** the generic identifier rule
  so they bind first.
- The `.` catch-all rule must be the **last** rule; it triggers
  `j0.lexErr` for unrecognized characters.
- The `\n` rule must bump the line counter via `j0.newline()` for
  error messages to point at the right place.

## Common bugs

- Forgetting to add a keyword to the lexer after adding it to the
  grammar — the parser will then see an `IDENTIFIER` and the rule
  silently misfires. See
  [`/wiki/chapters/ch03-lexical-analysis/exercises/edge-cases.md`](/wiki/chapters/ch03-lexical-analysis/exercises/edge-cases.md).
- Greedy string literal `\"[^\"]*\"` doesn't handle escaped quotes;
  this is on purpose for J0.
