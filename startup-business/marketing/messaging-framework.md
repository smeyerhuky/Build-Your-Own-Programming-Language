# Messaging Framework

---

## Core message

> **LangCraft makes programming language creation accessible to every developer.**

This is the single most important idea. Every piece of marketing content, every product description, every tweet should tie back to this idea.

---

## Recommended tagline

> **"Build the language you always wanted."**

**Why this tagline:**
- It speaks to the latent desire that both individual learners (I've always wondered how languages work) and enterprise builders (we need a DSL but it feels too hard) share.
- It is aspirational without being vague.
- It positions the product as an enabler, not a course — "build" implies creating something real.
- It avoids jargon (no "compilers," "parsers," "AST") that would alienate non-experts.

**Secondary taglines (for specific contexts):**

| Context | Tagline |
|---|---|
| Onboarding / signup page | "From zero to a running compiler in 10 minutes." |
| Enterprise landing page | "Ship a production DSL in weeks, not months." |
| Academic landing page | "The modern lab environment for your compilers course." |
| GitHub README | "Interactive compiler construction — browser-based, LLM-assisted." |

## Value propositions by persona

### Individual learner (Marcus)
1. **No setup.** Run the J0 compiler in a browser in 5 clicks. No JFlex install, no BYacc, no environment wars.
2. **Interactive, not passive.** Every concept is a runnable exercise with automated feedback — not a video.
3. **LLM help on demand.** When you're stuck at 11pm, the assistant knows the course and explains your error in plain English.

### University instructor (Dr. Priya)
1. **Consistent environment for every student.** No "it works on my machine" — everyone uses the same browser-based sandbox.
2. **Automated grading.** Exercises check themselves; progress dashboards show you who's stuck before they drop the course.
3. **Modern curriculum.** Java, VS Code, GitHub Actions — tools students will actually use in their careers.

### Enterprise DSL builder (Chen Wei)
1. **Full toolchain, not just a parser.** Lexer + parser + AST + type checker + code generator + VS Code plugin — all in one.
2. **Enterprise-ready.** SSO, audit logs, self-hosted option, SLA, DPA. Ready for your security team.
3. **Build on a proven foundation.** J0's architecture is the same as the internal DSLs powering companies like Uber, Google, and Stripe. Start there, adapt to your grammar.

### Indie builder (Yuki)
1. **Fast path from idea to running code.** Phase 1 to a working interpreter in a weekend.
2. **Share your work.** Portfolio link + LangCraft Showcase = a public home for your language project.
3. **Community.** Other language builders who care about the same things you do.

## Tone of voice

### What LangCraft sounds like

**Knowledgeable but approachable.** We are deep compiler experts who believe everyone can learn this. We never condescend. We never oversimplify.

**Direct and honest.** We say what the product does and doesn't do. We don't overpromise. If something is hard, we say it and explain how we make it less hard.

**Energetic but not hype-y.** We are excited about what we're building, but we avoid "revolutionary," "game-changing," "disruptive." The work speaks for itself.

**Specific over vague.** "Under 10 minutes to first build" beats "fast and easy." "23-opcode bytecode format" beats "simple." Numbers and specifics build trust.

### What LangCraft does not sound like

- ❌ "The world's most powerful compiler platform" (superlatives without proof)
- ❌ "Revolutionising the way developers build programming languages" (empty hype)
- ❌ "Simply click here to get started on your journey!" (patronising enthusiasm)
- ❌ "Our cutting-edge AI-powered solution leverages machine learning to..." (buzzword soup)
- ❌ "Building a compiler is easy with LangCraft!" (dishonest — it's hard; we make it learnable)

### Examples of on-brand copy

✅ "The J0 compiler pipeline runs in your browser. No installation. No setup. Just write J0 and see what happens."

✅ "When your parser fails, you want to know *why* — not read a cryptic error message and google for an hour. LangCraft's LLM assistant explains the error, points to the grammar rule that triggered it, and suggests a fix."

✅ "J0 is a small subset of Java. Small enough to implement in a semester. Real enough to teach every phase of a compiler."

## Messaging do's and don'ts

| Do | Don't |
|---|---|
| Cite specific numbers ($29/mo, <10 min, 5 phases) | Use vague claims ("fast," "affordable") |
| Lead with the benefit, not the feature | Lead with "we have a feature called..." |
| Acknowledge the hard parts honestly | Pretend compiler construction is easy |
| Use "you" — speak to the developer directly | Use "users" or "customers" |
| Reference the open-source foundation | Pretend LangCraft appeared from nowhere |
| Mention Clinton by name (credibility asset) | Bury the founder's credentials |
