# Deep dive — Phase 2: Repo anatomy

## Top-level layout

```
Build-Your-Own-Programming-Language/
├── ch3/       Lexer (Flex .l → Java scanner)
├── ch4/       Parser (Yacc .y → Java parser)
├── ch5/       AST builder (tree.java)
├── ch6/       Symbol tables (symtab.java)
├── ch7/       Type checker (typeinfo.java)
├── ch8/       Three-address code (tac.java)
├── ch9/       Stack-VM interpreter (j0machine.java)
├── ch11/      Bytecode serialization + loader
├── ch12/      Bytecode code generator (byc.java)
├── ch13/      x86-64 native code generator (x64.java)
├── startup/   This OKF onboarding bundle
├── wiki/      Deep-reference OKF bundle
└── README.md
```

`ch10`, `ch14`–`ch17` have no source directory — they are book-only
chapters. The wiki covers them in `wiki/chapters/ch10-*` etc.

## Inside a full chapter directory (example: ch3)

```
ch3/
├── javalex.l       ← source: hand-written Flex spec
├── nnws.l          ← source: warm-up Flex spec
├── simple.java     ← source: scanner driver
├── j0.java         ← source: full driver
├── parser.java     ← source: token-constant stub
├── token.java      ← source: token record
├── hello.java      ← source: J0 hello-world
├── makefile        ← build automation
└── (Yylex.java)    ← GENERATED by jflex; not committed
```

Generated files (`Yylex.java`, `parser.java` produced by BYacc,
`*.class`) are `.gitignore`d or simply absent from the repo. You
generate them locally during the build.

## Why the chapters share javalex.l by copy

Each chapter directory has its own copy of `javalex.l` (or imports
the Ch 3 version). This is intentional: later chapters evolve the
lexer (adding keywords for new statements). The copies diverge
intentionally rather than sharing a single file.

## The `makefile` pattern

Every chapter makefile follows the same pattern:

```makefile
# 1. Run the code generator (jflex / byacc)
Yylex.java: javalex.l
	jflex javalex.l

# 2. Compile Java sources
%.class: %.java Yylex.java
	javac $<

# 3. Default target
all: j0.class
```

Run `make` with no arguments to build the default target.
