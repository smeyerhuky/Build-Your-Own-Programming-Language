# Deep dive — Phase 4: The lexer–grammar change cycle

## The contract between lexer and grammar

`javalex.l` and `j0gram.y` are separate programs compiled by separate
tools, but they share a namespace of **token constants** via
`parser.java`. The contract:

```
javalex.l          parser.java          j0gram.y
---------          -----------          --------
return              int DO = 297;        %token DO
  j0.scan(                               DO: ...rule...
    parser.DO);
```

`parser.java` is auto-generated by BYacc from the `%token` declarations
in `j0gram.y`. JFlex reads `parser.java` to resolve `parser.DO`.

The update cycle for a new keyword:

```
1. Add  "do" { return j0.scan(parser.DO); }   in javalex.l
2. Add  %token DO                              in j0gram.y
3. run byacc -J j0gram.y   →  regenerates parser.java (with parser.DO)
4. run jflex javalex.l     →  regenerates Yylex.java (uses parser.DO)
5. run javac *.java        →  compiles everything
```

Steps 3 and 4 can be swapped; both must happen before `javac`.

## Single-character operator: no %token needed

`j0.ord("&")` returns the ASCII ordinal of `&` (38). The grammar
uses `'&'` (a character literal) rather than a `%token` name.

This works because single-character operators have unique ASCII values
that don't clash with any multi-character token constant (which BYacc
assigns values ≥ 256).

## What happens if you skip the %token declaration

JFlex fails to compile `Yylex.java` with:
```
cannot find symbol: variable DO in class parser
```

The error is in the JFlex-generated `Yylex.java`, not in your hand-written
files, which is why the message can look confusing.

## What happens if you put the keyword after {id}

The lexer compiles without error. At runtime, input `do { }` produces
`IDENTIFIER { }` instead of `DO { }`. The parser then reports a syntax
error at `{` — making it look like a parser bug when the real problem
is rule ordering in the lexer.

**CoT trace:** parser error at `{` → lexer returned wrong token for
`do` → `{id}` rule fires before `"do"` rule → move `"do"` above `{id}`.
